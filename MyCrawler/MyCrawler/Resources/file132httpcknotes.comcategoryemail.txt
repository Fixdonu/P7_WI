	<article id="post-497" class="post-497 post type-post status-publish format-standard hentry category-email category-smtp tag-email tag-headers tag-received tag-smtp">
		<header class="entry-header">
						<h1 class="entry-title"><a href="http://cknotes.com/email-received-headers/" rel="bookmark">Email &#8220;Received&#8221; Headers</a></h1>
			
						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/email-received-headers/" title="6:45 am" rel="bookmark"><time class="entry-date" datetime="2013-08-30T06:45:22+00:00">August 30, 2013</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>When an email is sent, it should contain no &#8220;Received&#8221; header fields.  Each time the email is processed by a mail server along the delivery route, a new Received header field is prepended to the email.  The very first Received header is added by the SMTP server that the SMTP client (such as your app or a program such as Outlook) connects to to send email.  </p>
<p>A typical scenario is that an email will contain two Received headers when delivery is complete.  The 1st is the Receive header added by your SMTP server.  This may often be something like &#8220;localhost&#8221; and may show an internal IP address, such as &#8220;192.168.1.###&#8221;.  Your SMTP server then relays the email to its destination, and the mail server at the destination (at the receiver) then adds its Received header. </p>
<p>The Received header has a syntax such as this:</p>
<pre>
        Received: from ? by ? via ? with ? id ? for ? ; date-time
</pre>
<p>Some mail servers may not include all the parts, and spammers often fake Received headers.</p>
					</div><!-- .entry-content -->

						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/mht-and-eml-are-both-mime/" title="7:06 am" rel="bookmark"><time class="entry-date" datetime="2010-06-21T07:06:45+00:00">June 21, 2010</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->

				<div class="entry-content">
			<p><strong>Question:</strong></p>
<p>I have added HTML email file support (thanks to your excellent  examples and library), but I have a question. It appears that it might  be easier for the end user to save their things they want to email as  a MHT file instead of HTML out of MS-Word. That way they have one file to  deal with. So is there a way to do the same method for email using and  MHT file?</p>
<p><strong>Answer:</strong></p>
<p>MHT and EML are the exact same thing &#8212; they are both MIME.   The only difference is the file extension and the intended use of the  file (as indicated by the file extension).  MHT is intended to be  displayed in Internet Explorer, and EML is intended to be an email.  A  .eml file will contain header files (Subject, To, From, etc.) that are  not present in MHT. Therefore, to &#8220;convert&#8221; an MHT to EML, simply load  it into an email object by calling LoadEml, set the Subject and From  properties.  Then Add recipients by calling AddTo, AddCC, etc., and then send&#8230;</p>

						<h1 class="entry-title"><a href="http://cknotes.com/understanding-bcc-blind-carbon-copy/" rel="bookmark">Understanding BCC (Blind Carbon Copy)</a></h1>
			
						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/understanding-bcc-blind-carbon-copy/" title="1:52 pm" rel="bookmark"><time class="entry-date" datetime="2010-02-18T13:52:19+00:00">February 18, 2010</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>When an email is sent with BCC recipients, the BCC email addresses are not listed in the header.   The &#8220;To&#8221; recipients are listed in the &#8220;To&#8221; header field, and the &#8220;CC&#8221; recipients are listed in the &#8220;CC&#8221; header field, but the BCC recipients are intentionally left out.  This is what makes it a &#8220;blind&#8221; carbon copy &#8212; the recipients are unable to see the BCC recipients.  (It is by design.)  Otherwise the BCC recipients would be equivalent to &#8220;CC&#8221; recipients.  To understand how BCC recipients receive an email, I recommend reading this blog post: <a title="SMTP Protocol Explained" href="http://cknotes.com/?p=40">SMTP Protocol in a Nutshell</a>.   The BCC recipients are passed to the SMTP server in &#8220;RCPT TO&#8221; commands.</p>

	<article id="post-285" class="post-285 post type-post status-publish format-standard hentry category-email tag-email">
		<header class="entry-header">
						<h1 class="entry-title"><a href="http://cknotes.com/appending-to-an-email-body/" rel="bookmark">Appending to an email body</a></h1>
			
						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/appending-to-an-email-body/" title="9:36 am" rel="bookmark"><time class="entry-date" datetime="2010-02-17T09:36:23+00:00">February 17, 2010</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>Here is a C# example to append more text to an email&#8217;s body:</p>
<pre>
    Chilkat.Email email = new Chilkat.Email();

    bool success = email.LoadEml("in.eml");
    if (!success)
    {
        textBox1.Text = email.LastErrorText;
        return;
    }

    // Does this email have a plain-text body?
    if (email.HasPlainTextBody())
    {
        string s = email.GetPlainTextBody();
        s += "\r\nMore text appended to the email's plain-text body.";
        // Replace the plain-text body:
        if (email.HasHtmlBody())
        {
            // This is an email with both plain-text and HTML alternatives
            // The AddPlainTextAlternative body replaces the plain-text body if it already exists.
            email.AddPlainTextAlternativeBody(s);
        }
        else
        {
            // This is a plain-text only email.
            email.Body = s;
        }
    }

    if (email.HasHtmlBody())
    {
        string s = email.GetHtmlBody();
        s += "&lt;p>More text appended to the email's plain-text body.&lt;/p>";
        // Replace the HTML body:
        if (email.HasPlainTextBody())
        {
            // This is an email with both plain-text and HTML alternatives
            // The AddHtmlAlternativeBody replaces the HTML body if it already exists.
            email.AddHtmlAlternativeBody(s);
        }
        else
        {
            // This is an HTML-only email.
            email.SetHtmlBody(s);
        }
    }

    email.SaveEml("out.eml");
</pre>

	<article id="post-280" class="post-280 post type-post status-publish format-standard hentry category-attachments category-email tag-attachments tag-email">
		<header class="entry-header">
						<h1 class="entry-title"><a href="http://cknotes.com/email-attachment-not-found/" rel="bookmark">Email Attachment Not Found?</a></h1>
			
						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/email-attachment-not-found/" title="6:29 am" rel="bookmark"><time class="entry-date" datetime="2010-02-09T06:29:25+00:00">February 9, 2010</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>The following is a list of the most common mistakes made when expecting to find an attachment in an email object:</p>
<ol>
<li>You downloaded headers-only.  If you call a method to download only the headers of an email, then the attachments will not be included.  With IMAP you will still get information about the attachments, but with POP3 you will not get them at all.  See this blog post about <a title="POP3 headers and attachments" href="http://cknotes.com/?p=85">POP3 email headers and attachment info</a>.</li>
<li>What you are expecting to see as an attachment is actually a &#8220;related item&#8221;.  A related item is an image, style sheet, etc. that is part of the HTML presentation.  It is an embedded item that is referenced by the HTML body of an email and is not a formal attachment.  You&#8217;ll find a complete set of properties and methods for related items:  NumRelatedItems, GetRelatedData, SaveRelatedItem, etc.</li>
<li>It is an &#8220;attached message&#8221;.  Similar to related items, a complete email that is attached to another email is considered separately by Chilkat.  The property is NumAttachedMessages.  To get the Nth attached message, call GetAttachedMessage(index).  this returns a new email object that is the Nth attached message.</li>
<li>It is an attachment found within an attached message.  You would first need to get the attached message and then you&#8217;ll see the attachments within it.</li>
</ol>
