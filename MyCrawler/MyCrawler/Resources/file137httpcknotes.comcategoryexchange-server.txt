						<h1 class="entry-title"><a href="http://cknotes.com/imap-login-bad-command-received-in-invalid-state-2/" rel="bookmark">IMAP Login: BAD Command received in Invalid state.</a></h1>
			
						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/imap-login-bad-command-received-in-invalid-state-2/" title="5:23 am" rel="bookmark"><time class="entry-date" datetime="2010-08-19T05:23:23+00:00">August 19, 2010</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>This error can happen if you connect to an IMAP server without SSL/TLS (i.e. at port 143) but the server does not allow unsecure sessions.  Oddly enough, the Microsoft Exchange IMAP4 server will accept the connection, but will fail any login attempt with this error.  Here is a sample SessionLog:</p>
<pre>
Connecting to IMAP server at xxxx.com:143
> ----IMAP RESPONSE----
> * OK The Microsoft Exchange IMAP4 service is ready.
> ----IMAP REQUEST----
> aaab LOGIN "mylogin"
> ----IMAP RESPONSE----
> aaab BAD Command received in Invalid state.
</pre>
<p>The solution is to use port 993 w/ SSL/TLS.  Do this by setting the Chilkat.Imap.Port property = 993, and the Chilkat.Imap.Ssl property = True prior to connecting.</p>

						<h1 class="entry-title"><a href="http://cknotes.com/imap-login-bad-command-received-in-invalid-state/" rel="bookmark">IMAP Login &#8211; BAD Command received in Invalid state</a></h1>
			
						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/imap-login-bad-command-received-in-invalid-state/" title="7:49 am" rel="bookmark"><time class="entry-date" datetime="2009-10-19T07:49:25+00:00">October 19, 2009</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>A Chilkat customer experienced the following behavior with Exchange Server in response to an IMAP login.  This is the (edited) contents of the IMAP object&#8217;s SessionLog property:</p>
<pre>----INFO----
Connecting to IMAP server at ***:143
----IMAP RESPONSE----
* OK The Microsoft Exchange IMAP4 service is ready.
----IMAP REQUEST----
aaab LOGIN "***"
----IMAP RESPONSE----
aaab BAD Command received in Invalid state.
</pre>
<p><strong>Solution:</strong><br />
The IMAP server&#8217;s response is misleading.</p>
<p>In this case, the Exchange Server only allows SSL by default, so it was necessary to connect to port 993 with SSL on.   Also, the login name needed to be domain\username.</p>
<p>Here is more information: <a href="http://www.chilkatsoft.com/p/p_175.asp">IMAP Exchange Server Login</a>.</p>

						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/sent-email-not-found-in-exchanges-sent-items-folder/" title="3:31 pm" rel="bookmark"><time class="entry-date" datetime="2009-09-21T15:31:09+00:00">September 21, 2009</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p><strong>Question:</strong></p>
<p>After I send an email using Chilkat, I don&#8217;t see the email in the &#8220;Sent Items&#8221; folder in Exchange.  How can this be done?  Is it possible?</p>
<p><strong>Answer:</strong></p>
<p>The &#8220;Sent Items&#8221; folder is specific to Exchange/Outlook &#8212; it is not part of the SMTP protocol. You may use Chilkat IMAP to &#8220;append&#8221; a copy of the sent email to an Exchange folder (assuming the Exchange Server&#8217;s IMAP protocol is enabled).   You would do this by calling Chilkat IMAP&#8217;s AppendMail or AppendMime method after the email has been successfully sent.</p>

						<h1 class="entry-title"><a href="http://cknotes.com/exchange-server-pop3-imap-and-smtp/" rel="bookmark">Exchange Server &#8211; POP3, IMAP, and SMTP</a></h1>
			
						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/exchange-server-pop3-imap-and-smtp/" title="6:01 pm" rel="bookmark"><time class="entry-date" datetime="2009-09-15T18:01:59+00:00">September 15, 2009</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p><strong>Question:</strong></p>
<p>May I ask if is possible use Chilkat components to connect to MS Exchange 6.5 and higher ?<br />
I am thinking about purchase of MailMan component or other as it is, however I would like read some example in C# for reading emails from Exchange and sending emails via Exchange<br />
after connecting to the specified account domain/logname &amp; password.<br />
May I see some examples somewhere on your site?<br />
May you recommend correct components please ?</p>
<p><strong>Answer:</strong></p>
<p>Exchange Server (including v6.5 and higher) implements the server-side of the POP3, IMAP, and SMTP protocols.  Therefore, any software that implements the client-side of these protocols will work with Exchange Server in exactly the same way as any other email server. (See the note below regarding NTLM authentication.)</p>
<p>The purpose of a protocol is to allow any client-side implementation to communicate with any server-side implementation.  As long as each side correctly implements the protocol to specification, everything works.  For example, a web browser implements the client-side of the HTTP protocol.  You may not realize it, but it communicates with many different web server implementations: IIS, Apache, etc.  You can browse the web because the both clients (i.e. web browsers) and servers correctly implement the HTTP protocol.</p>
<p>Therefore, all of the examples involving POP3, SMTP, and IMAP at <a title="Example Code" href="http://www.example-code.com">http://www.example-code.com</a> are examples that will work with Exchange Server.</p>
<p>The Chilkat Email license ($99 for a 1-Developer license) is what you would need for the POP3 and SMTP functionality.  The Chilkat IMAP license ($199 for a 1-Developer license) includes all three: POP3, SMTP, and IMAP.  Note: 1-Developer does not mean &#8220;1-Computer&#8221;.  It simply means that one person may develop programs using the component.  All Chilkat licenses allow for royalty-free redistribution, meaning you may use the component/library on any number of computers.</p>
<p>*** MS Exchange implements an additional authentication mechanism called &#8220;NTLM&#8221;, which is also known as Windows Integrated Authentication.   Chilkat supports NTLM for POP3, IMAP, and SMTP.</p>
