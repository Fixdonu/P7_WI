						<h1 class="entry-title"><a href="http://cknotes.com/c-language-callbacks/" rel="bookmark">C Language Callbacks</a></h1>
			
						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/c-language-callbacks/" title="7:17 am" rel="bookmark"><time class="entry-date" datetime="2016-05-21T07:17:46+00:00">May 21, 2016</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>This example demonstrates the general pattern for implementing some standard event callbacks functions for the Chilkat &#8220;C&#8221; API.  All Chilkat &#8220;classes&#8221; what have events will use the standard Chilkat events shown in the example below (AbortCheck, PercentDone, and ProgressInfo).</p>
<p>This example demonstrates callbacks for an SFTP download, but the same technique applies to any other Chilkat &#8220;class&#8221;, such as HTTP, FTP2, MailMan, Rest, etc.</p>
<pre class="brush: cpp; title: ; notranslate" title="">
#include &lt;stdio.h&gt;

#include &lt;C_CkSFtp.h&gt;

BOOL myAbortCheck()
    {
    // To abort the current Chilkat method call, return non-zero.
    return 0;
    }

BOOL myPercentDone(int pctDone)
    {
    printf(&quot;Percent Done: %d\n&quot;,pctDone);

    // To abort the current Chilkat method call, return non-zero.
    return 0;
    }

void myProgressInfo(const char *name, const char *value)
    {
    printf(&quot;%s: %s\n&quot;,name,value);
    }

void ChilkatSample(void)
    {
    HCkSFtp sftp;
    BOOL success;
    int port;
    const char *hostname;
    const char *handle;

    // Declare event callbacks.
    BOOL (*fnAbortCheck)() = myAbortCheck;
    BOOL (*fnPercentDone)(int pctDone) = myPercentDone;
    void (*fnProgressInfo)(const char *name, const char *value) = myProgressInfo;

    sftp = CkSFtp_Create();

    //  Any string automatically begins a fully-functional 30-day trial.
    success = CkSFtp_UnlockComponent(sftp,&quot;Anything for 30-day trial&quot;);
    if (success != TRUE) {
        printf(&quot;%s\n&quot;,CkSFtp_lastErrorText(sftp));
        CkSFtp_Dispose(sftp);
        return;
    }

    // Setup event callbacks.
    CkSFtp_setAbortCheck(sftp, fnAbortCheck);
    CkSFtp_setPercentDone(sftp, fnPercentDone);
    CkSFtp_setProgressInfo(sftp, fnProgressInfo);

    // Make sure to set a HeartbeatMs for AbortCheck / PercentDone callbacks..
    // Set the HeartbeatMs to 250 milliseconds.
    CkSFtp_putHeartbeatMs(sftp,250);

    //  Set some timeouts, in milliseconds:
    CkSFtp_putConnectTimeoutMs(sftp,5000);
    CkSFtp_putIdleTimeoutMs(sftp,10000);

    //  Connect to the SSH server.
    //  The standard SSH port = 22
    //  The hostname may be a hostname or IP address.

    hostname = &quot;my-Sftp-Server-Domain-Or-IPAddress&quot;;
    port = 22;
    success = CkSFtp_Connect(sftp,hostname,port);
    if (success != TRUE) {
        printf(&quot;%s\n&quot;,CkSFtp_lastErrorText(sftp));
        CkSFtp_Dispose(sftp);
        return;
    }

    //  Authenticate with the SSH server.  Chilkat SFTP supports
    //  both password-based authenication as well as public-key
    //  authentication.  This example uses password authenication.
    success = CkSFtp_AuthenticatePw(sftp,&quot;myLogin&quot;,&quot;myPassword&quot;);
    if (success != TRUE) {
        printf(&quot;%s\n&quot;,CkSFtp_lastErrorText(sftp));
        CkSFtp_Dispose(sftp);
        return;
    }

    //  After authenticating, the SFTP subsystem must be initialized:
    success = CkSFtp_InitializeSftp(sftp);
    if (success != TRUE) {
        printf(&quot;%s\n&quot;,CkSFtp_lastErrorText(sftp));
        CkSFtp_Dispose(sftp);
        return;
    }

    //  Open a file on the server:
    handle = CkSFtp_openFile(sftp,&quot;hamlet.xml&quot;,&quot;readOnly&quot;,&quot;openExisting&quot;);
    if (CkSFtp_getLastMethodSuccess(sftp) != TRUE) {
        printf(&quot;%s\n&quot;,CkSFtp_lastErrorText(sftp));
        CkSFtp_Dispose(sftp);
        return;
    }

    //  Download the file:
    success = CkSFtp_DownloadFile(sftp,handle,&quot;c:/temp/hamlet.xml&quot;);
    if (success != TRUE) {
        printf(&quot;%s\n&quot;,CkSFtp_lastErrorText(sftp));
        CkSFtp_Dispose(sftp);
        return;
    }

    //  Close the file.
    success = CkSFtp_CloseHandle(sftp,handle);
    if (success != TRUE) {
        printf(&quot;%s\n&quot;,CkSFtp_lastErrorText(sftp));
        CkSFtp_Dispose(sftp);
        return;
    }

    printf(&quot;Success.\n&quot;);


    CkSFtp_Dispose(sftp);

    }

int main()
{
  ChilkatSample();
  return 0;
}
</pre>

	<article id="post-656" class="post-656 post type-post status-publish format-standard hentry category-asynchronous category-events category-java tag-async tag-asynchronous tag-callbacks tag-events tag-java">
		<header class="entry-header">
						<h1 class="entry-title"><a href="http://cknotes.com/event-callbacks-in-java/" rel="bookmark">Event Callbacks in Java</a></h1>
			
						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/event-callbacks-in-java/" title="5:11 am" rel="bookmark"><time class="entry-date" datetime="2015-08-22T05:11:08+00:00">August 22, 2015</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>Event callbacks in Java (including Android) are supported starting in v9.5.0.52.   To receive event callbacks, first create a Java class derived from one of the Chilkat event callback classes.  The event callback classes are: CkBaseProgress, CkHttpProgress, CkZipProgress, CkFtp2Progress, CkMailManProgress, CkTarProgress, and CkSFtpProgress.</p>
<p>All future Chilkat classes will only use CkBaseProgress.  Theses event callback classes will be documented in the &#8220;Events&#8221; section of each class&#8217;s online reference documentation.</p>
<p>Here is an example of an HTTP download using event callbacks in Java.  First we have the MyHttpProgress derived from CkHttpProgress.  Following that, we have the simple Java program that installs the event handler and does the HTTP download.  Event callbacks will also happen with the asynchronous version of the methods, but beware of the fact that when asynchronous, the task is running in a background thread and the callback occurs in the background thread.  </p>
<pre class="brush: java; title: ; notranslate" title="">
import com.chilkatsoft.CkHttpProgress;
import com.chilkatsoft.CkTask;


public class MyHttpProgress extends CkHttpProgress 
{	
  public boolean AbortCheck()
  	{
      System.out.println(&quot;AbortCheck&quot;);
      // Return true to abort, false to allow the method to continue.
      return false;
  	}
  	
  // pctDone is a value from 0 to 100
  // (it is actually value from 0 to the PercentDoneScale property setting)
  public boolean PercentDone(int pctDone)
  {
    System.out.println(pctDone);
    // Return true to abort, false to allow the method to continue.
    // Note: A PercentDone event is the equivalent of an AbortCheck.  
    // When PercentDone events are frequently firing, AbortCheck events are suppressed.
    // AbortCheck events will fire when the time between PercentDone events is longer 
    // than the HeartbeatMs property setting.
    return false;
  }
  
  public void ProgressInfo(String name, String value)
  {
    System.out.println(name + &quot;: &quot; + value);
  }
  
  public void TaskCompleted(CkTask task)
  {
     System.out.println(&quot;task completed!&quot;);
  }
  
  }
</pre>
<p><strong>The synchronous Java program w/ event callbacks&#8230; (the asynchronous version follows..)</strong></p>
<pre class="brush: java; title: ; notranslate" title="">
import com.chilkatsoft.CkHttp;

public class HttpWithEvents {
	
  static {
    try {
    	
    	System.loadLibrary(&quot;chilkat&quot;);
    	
    } catch (UnsatisfiedLinkError e) {
      System.err.println(&quot;Native code library failed to load.\n&quot; + e);
      System.exit(1);
    }
  }


	
  public static void main(String argv[])
  	{
    CkHttp http = new CkHttp();
    
    // Install an event callback handler to get progress events.
    MyHttpProgress myProgress = new MyHttpProgress();
    http.put_EventCallbackObject(myProgress);
    
    boolean success;

    //  Any string unlocks the component for the 1st 30-days.
    success = http.UnlockComponent(&quot;Anything for 30-day trial&quot;);
    if (success != true) {
        System.out.println(http.lastErrorText());
        return;
    }

    //  Download a file at a URL.
    success = http.Download(&quot;http://www.chilkatsoft.com/download/9.5.0.51/ChilkatDotNet45-9.5.0-x64.zip&quot;,&quot;ChilkatDotNet45-9.5.0-x64.zip&quot;);
    if (success != true) {
        System.out.println(http.lastErrorText());
        return;
    }
    
    System.out.println(&quot;OK&quot;);
  	}
  }
</pre>
<p><strong>The asynchronous Java program w/ event callbacks&#8230;</strong></p>
<pre class="brush: java; title: ; notranslate" title="">
import com.chilkatsoft.CkHttp;
import com.chilkatsoft.CkTask;

public class AsyncHttp {
	
  static {
    try {
    	
    	System.loadLibrary(&quot;chilkat&quot;);
    	
    } catch (UnsatisfiedLinkError e) {
      System.err.println(&quot;Native code library failed to load.\n&quot; + e);
      System.exit(1);
    }
  }


	
  public static void main(String argv[])
  	{
    CkHttp http = new CkHttp();
    
    // Install an event callback handler to get progress events.
    MyHttpProgress myProgress = new MyHttpProgress();
    http.put_EventCallbackObject(myProgress);
    
    boolean success;

    //  Any string unlocks the component for the 1st 30-days.
    success = http.UnlockComponent(&quot;Anything for 30-day trial&quot;);
    if (success != true) {
        System.out.println(http.lastErrorText());
        return;
    }

    //  Download a file at a URL.
    CkTask task = http.DownloadAsync(&quot;http://www.chilkatsoft.com/download/9.5.0.51/ChilkatDotNet45-9.5.0-x64.zip&quot;,&quot;ChilkatDotNet45-9.5.0-x64.zip&quot;);
    if (success != true) {
        System.out.println(http.lastErrorText());
        return;
    }
    
    task.put_UserData(&quot;chilkatDotNet45&quot;);
    
    if (!task.Run()) {
        System.out.println(task.lastErrorText());
        return;
	}
  
    System.out.println(&quot;OK, task is running...&quot;);
    
    // Wait a max of 10 seconds for it to finish.
    success = task.Wait(10000);
    
    // What is the task status?
    System.out.println(&quot;task status = &quot; + task.status());
  	}
  }
</pre>

						<h1 class="entry-title"><a href="http://cknotes.com/delphi-activex-event-handling/" rel="bookmark">Delphi ActiveX Event Handling</a></h1>
			
						<div class="entry-meta">
				<span class="sep">Posted on </span><a href="http://cknotes.com/delphi-activex-event-handling/" title="8:25 am" rel="bookmark"><time class="entry-date" datetime="2012-02-17T08:25:19+00:00">February 17, 2012</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->
			
					</header><!-- .entry-header -->

				<div class="entry-content">
			<p>ActiveX event callbacks can be used to get progress information for lengthy operations, especially for API calls involving communications with servers, such as FTP, SFTP (SSH), IMAP, POP3, SMTP, HTTP, etc.  </p>
<p>To use an ActiveX in Delphi, the component must be &#8220;imported&#8221;.  This generates wrapper files (TLB.pas) in the &#8220;imports&#8221; directory, and may also install icons on the component palettte.  </p>
<p>The Chilkat components are non-visual.  There are two ways of  creating an instance of a Chilkat object: (1) drag and drop the icon from the palette onto the form.  The non-visual object will be seen as a small box on the form.  The component  may be selected, and the events can be obtained from the object inspector.  (2) The <strong>recommended</strong> technique for instantiating a Chilkat object is to do so dynamically (do not drag and drop onto the form).  In this case, you can go to the generated TLB.pas file and locate the event signatures (i.e. functions).  Write your own method that has the same function signature, and point the relevant event property to your new method.</p>
<p>For example, here are the event signatures in the TLB.pas for Chilkat FTP2:</p>
<pre>
  TChilkatFtp2PutProgress = procedure(ASender: TObject; pctDone: Integer) of object;
  TChilkatFtp2GetProgress = procedure(ASender: TObject; pctDone: Integer) of object;
  TChilkatFtp2AbortCheck = procedure(ASender: TObject; out abort: Integer) of object;
  TChilkatFtp2BeginDownloadFile = procedure(ASender: TObject; const path: WideString; 
                                                              out skip: Integer) of object;
  TChilkatFtp2EndDownloadFile = procedure(ASender: TObject; const path: WideString; 
                                                            numBytes: Integer) of object;
  TChilkatFtp2VerifyDownloadDir = procedure(ASender: TObject; const path: WideString; 
                                                              out skip: Integer) of object;
  TChilkatFtp2BeginUploadFile = procedure(ASender: TObject; const path: WideString; 
                                                            out skip: Integer) of object;
  TChilkatFtp2EndUploadFile = procedure(ASender: TObject; const path: WideString; numBytes: Integer) of object;
  TChilkatFtp2VerifyUploadDir = procedure(ASender: TObject; const path: WideString; 
                                                            out skip: Integer) of object;
  TChilkatFtp2VerifyDeleteDir = procedure(ASender: TObject; const path: WideString; 
                                                            out skip: Integer) of object;
  TChilkatFtp2VerifyDeleteFile = procedure(ASender: TObject; const path: WideString; 
                                                             out skip: Integer) of object;

</pre>

				<div class="entry-content">
			<p><a title="Chilkat v9.3.0 Release Notes" href="http://cknotes.com/?p=391">Chilkat v9.3.0</a> for Cocoa now includes the asynchronous functionality and event callback functionality that has been available in the C++ libs.</p>
<p>Asynchronous methods run in a background thread such that the call immediately returns to the application.  <span style="color: #ff0000;">Events are not fired w/ asynchronous method calls</span> &#8212; your app would     periodically check to see if the background task is finished.</p>
<p>To implement an event callback, your app would define/implement a class that     inherits from one of the Cko*Progress base classes.  You provide     method implementations to override the methods specified in the     Cko*Progress classes (see the header files).  The callback object is     specified via the EventCallbackObject property.  For example:</p>
<p>// property setter: EventCallbackObject<br />
&#8211; (void)setEventCallbackObject: (CkoFtp2Progress *)eventObj;</p>

				<span class="sep">Posted on </span><a href="http://cknotes.com/ftp-progress-monitoring/" title="12:13 pm" rel="bookmark"><time class="entry-date" datetime="2010-07-21T12:13:54+00:00">July 21, 2010</time></a><span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="http://cknotes.com/author/admin/" title="View all posts by admin" rel="author">admin</a></span></span>			</div><!-- .entry-meta -->

				<div class="entry-content">
			<p>This post points to the various examples and blog posts scattered among the chilkatsoft.com, example-code.com, and cknotes.com involving progress monitoring of&nbsp; FTP uploads and downloads using the Chilkat FTP2 ActiveX, .NET component, and C++ libs.</p>
<p><strong>Important for Monitoring Upload Progress: </strong>The SendBufferSize property is set to a large value (512K) to maximize performance.&nbsp; Unfortunately, this usually ruins the frequency of &#8220;percent done&#8221; callbacks for upload (not downloads, just uploads).&nbsp; To improve the upload &#8220;percent done&#8221; callback frequency, set the SendBufferSize property to a smaller value, such as 4K.&nbsp; It may slow the upload transfer rate a bit, but it improves the progress monitoring capability.&nbsp; You can experiment with different values for SendBufferSize to determine what best suits your needs.</p>
<p><strong>Monitoring Download Percent-Done: </strong>Be sure to read about the AutoGetSizeForProgress and the ProgressMonSize properties in the Chilkat online reference documentation.&nbsp; Depending on your FTP server, it may be necessary to set the AutoGetSizeForProgress equal to True to properly receive percent-done event callbacks.&nbsp; In most cases, neither property is needed &#8212; it entirely depends on whether the FTP server provides file-size information in the intermediate response to a &#8220;RETR&#8221; FTP protocol command.</p>
